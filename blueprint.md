# Приложение для управления уборкой гостиничных номеров

## Обзор

Этот документ описывает план и архитектуру React-приложения, предназначенного для управления уборкой гостиничных номеров. Приложение использует Redux Toolkit для сложного управления состоянием и Material-UI для пользовательского интерфейса. Цель — создать масштабируемое, типобезопасное и удобное в поддержке приложение, которое автоматизирует распределение задач по уборке и уведомляет о результатах.

## Основные сущности

### Номер (Room)
- **id**: Уникальный идентификатор (например, '101').
- **floor**: Этаж (1–5).
- **number**: Номер на этаже (1–20).
- **capacity**: Вместимость номера (условные единицы трудозатрат).
- **status**: Состояние (`clean` – чистый, `dirty` – требует уборки, `assigned` – назначен горничной).
- **assignedTo**: ID горничной, которой назначен номер.

### Горничная (Housekeeper)
- **id**: Уникальный идентификатор (например, 'h1').
- **name**: Имя горничной.
- **workloadCapacity**: Максимальная загрузка (сумма capacity номеров).
- **assignedRooms**: Массив ID номеров, назначенных ей.
- **color**: Персональный цвет для визуальной идентификации в UI.

## Логика распределения

При поступлении запроса на уборку система автоматически распределяет грязные номера между горничными, оптимизируя их маршруты.

1.  **Группировка по этажам**: В первую очередь, все номера, требующие уборки (`status: 'dirty'`), сортируются по номеру этажа. Это гарантирует, что алгоритм будет обрабатывать номера последовательно, этаж за этажом, минимизируя перемещения горничных по гостинице.
2.  **Сортировка горничных по загрузке**: Для каждого грязного номера система динамически сортирует горничных по их текущей загрузке (от меньшей к большей). Это позволяет всегда выбирать наиболее свободного сотрудника.
3.  **Выбор оптимальной горничной**: Выбирается первая горничная из отсортированного списка, чья оставшаяся вместимость (`workloadCapacity` минус `currentWorkload`) достаточна для уборки данного номера.
4.  **Назначение и обновление статуса**: Если подходящая горничная найдена, номер назначается ей, его статус меняется на `assigned`, и текущая загрузка горничной обновляется.
5.  **Очередь на будущее**: Если для номера не нашлось свободной горничной, он остается в статусе `dirty` и будет распределен в следующем цикле, когда ресурсы освободятся.

## Интерактивность и Уведомления

- **Изменение статуса номера**: Пользователи могут изменять статус номера между «чистый» и «грязный», кликая на карточку номера. Это действие недоступно, если номер уже назначен горничной.
- **Автоматическое распределение**: Пользователь может нажать кнопку для запуска алгоритма распределения грязных номеров.
- **Сброс состояния**: Пользователь может нажать кнопку «Сбросить», чтобы вернуть все номера в состояние «чистый» и очистить все назначения у горничных.
- **Уведомления в Telegram**: После каждого распределения номеров приложение автоматически отправляет отчет в указанный Telegram-канал. В сообщении содержится информация о том, какие номера какой горничной были назначены. Для этой функции используются переменные окружения `REACT_APP_TELEGRAM_BOT_TOKEN` и `REACT_APP_TELEGRAM_CHANNEL_ID`.

## Стилизация и UI

### Цветовое кодирование
- **Статус номера "Грязный"**: `#ffebee` (светло-красный)
- **Статус номера "Чистый"**: `#e8f5e9` (светло-зеленый)
- **Назначенные номера и горничные**: Каждой горничной присвоен уникальный цвет. Когда номер назначается горничной, его карточка окрашивается в ее цвет для легкой визуальной идентификации. Список горничных также использует эти цвета.

### Визуальная согласованность
- **Карточки номеров**: Все карточки номеров имеют одинаковую высоту для создания аккуратного и единого вида. Это достигается за счет резервирования места для имени горничной, даже если номер еще не назначен.

## План реализации

1.  **Определение типов**: Интерфейсы `Room` и `Housekeeper` определены в `src/types/index.ts`.
2.  **Настройка Redux Store**: Настроен Redux store в `src/app/store.ts`.
3.  **Реализация Redux Slices**:
    *   `roomsSlice`: Управляет списком номеров и их статусами, включает редьюсер `resetRooms`.
    *   `housekeepersSlice`: Управляет списком горничных, включает редьюсер `resetAssignments`.
    *   `assignmentsSlice`: Реализует логику распределения номеров с учетом этажности и отправляет уведомление в Telegram.
4.  **Создание UI компонентов**:
    *   `RoomsList`: Отображает номера с одинаковой высотой карточек, используя цвета для назначенных комнат.
    *   `HousekeepersList`: Отображает горничных, подсвечивая каждую ее цветом.
    *   `AssignmentPanel`: Содержит кнопки для запуска распределения и сброса состояния.
5.  **Интеграция и стилизация**: Компоненты собраны в `App.tsx`. Material-UI используется для создания интерфейса.
